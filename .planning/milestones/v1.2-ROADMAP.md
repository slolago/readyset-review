## Milestone Summary

**Key Decisions:**
- Token as Firestore document ID for review links — strongly-consistent direct doc lookup
- Named exports (not default) for UI components to match Button.tsx / Spinner.tsx convention
- Upload thumbnail via server-side route to avoid GCS CORS issues
- `application/x-frame-move` MIME type for drag payload to distinguish internal drags from OS drops
- GCS dual signed URL strategy — `signedUrl` (inline playback) + `downloadUrl` (attachment disposition)
- `createPortal` to `document.body` for Dropdown.tsx to escape overflow-hidden ancestors
- `hideActions` prop on AssetCard for guest read-only enforcement on review links
- Tab bar uses `-mb-px` connected underline pattern established in Phase 9, reused in Phases 14 and 21
- `db.getAll()` batch lookup for admin projects owner join — avoids N+1 Firestore reads
- 250ms debounce + `versionRef` stale-response guard for UserSearchCombobox

**Tech Debt Incurred:**
- Phases 16-18 completed via git commits outside GSD (no PLAN/SUMMARY artifacts)
- UserSearchCombobox name search is case-sensitive — `nameLower` field strategy deferred to v2
- `generateToken()` in utils.ts is a dead export — cleanup candidate
- Review page download fallback to `signedUrl` may open in browser tab if GCS attachment URL generation fails (documented acceptable edge case)

**Issues Resolved:**
- Review link 404 immediately after creation (token as doc ID fix)
- Upload stuck at "Uploading..." for subsequent uploads
- Three-dot menu z-index issues on review page (createPortal fix)
- Dashboard showing 0 stats (Firestore collectionGroup queries)
- VU meter zero readings (Web Audio API direct-to-video element wiring)
