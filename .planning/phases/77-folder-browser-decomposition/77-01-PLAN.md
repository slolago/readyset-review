  <files>src/hooks/useProject.ts</files>
  <action>
    Edit `src/hooks/useProject.ts`:

    The mount effect at lines 54-56 currently only fires `fetchProject()`:

    ```typescript
    useEffect(() => {
      fetchProject();
    }, [fetchProject]);
    ```

    Change it to fire both `fetchProject()` AND `fetchFolders(null)` in parallel via `Promise.all`:

    ```typescript
    useEffect(() => {
      // PERF-22: fire project + root folders in parallel, not serially.
      // Both setters are independent — a rejection in one MUST NOT cancel the other.
      Promise.all([fetchProject(), fetchFolders(null)]);
    }, [fetchProject, fetchFolders]);
    ```

    Notes — follow CLAUDE.md Surgical Changes rule:
    - `fetchProject` already has internal try/catch/finally that sets `error` / clears `loading`. Preserve semantics.
    - `fetchFolders` already has internal try/catch that console.errors. Preserve semantics.
    - Because both functions swallow their own errors internally, `Promise.all` will never reject — no extra `.catch` is required and no behavior change on error paths.
    - Do NOT await the `Promise.all` in the effect body — it's fire-and-forget by design (the effect itself is sync). Both callbacks update state via their own setters.
    - Include `fetchFolders` in the dep array.
    - Do NOT touch the `fetchProject` / `fetchFolders` callback definitions — leave them intact so external callers (FolderBrowser.tsx refetches after mutations) still work.
    - Do NOT remove the returned `fetchFolders` from the hook's return object — it's used externally for post-mutation refreshes.

    Scope: ~5 line change (the useEffect body + dep array).
  </action>
