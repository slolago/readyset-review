Make exactly four changes to `src/app/review/[token]/page.tsx`:

**1. Add useAuth import (top of file, after existing imports):**

```typescript
import { useAuth } from '@/hooks/useAuth';
```

**2. Destructure auth state inside ReviewPage() (after the existing useState declarations, before fetchReview):**
```typescript
const { user, loading: authLoading } = useAuth();
```

**3. Add useEffect to auto-populate guestInfo from auth (after the existing useEffect for fetchReview, around line 68):**
```typescript
useEffect(() => {
  if (!authLoading && user) {
    setGuestInfo({ name: user.name, email: user.email });
  }
}, [authLoading, user]);
```

This effect:
- Waits for Firebase auth to resolve (`!authLoading`)
- If user is logged in, overwrites guestInfo with their real identity
- Auth identity always wins over localStorage (handles case where returning guest later signed up)
- Does NOT guard on `!guestInfo` -- an auth user's identity must take precedence over any stale localStorage guest name

**4. Update the guest form gate condition (line 245 area):**

Change:
```typescript
if (!guestInfo && data.reviewLink.allowComments) {
```
To:
```typescript
if (!guestInfo && data.reviewLink.allowComments && !authLoading) {
```

This prevents the guest form from flashing during the brief window when Firebase auth is still resolving. While `authLoading` is true, the page will stay on the loading spinner (which is already shown while `data` is being fetched). Once auth resolves, if user is logged in the useEffect populates guestInfo and the gate is skipped entirely. If user is not logged in, guestInfo remains null and the guest form shows normally.

**What NOT to change:**
- Do NOT touch ReviewGuestForm component
- Do NOT touch handleGuestSubmit function
- Do NOT remove the localStorage initialiser in useState -- it still serves non-logged-in returning guests
- Do NOT add a separate isLoggedIn state -- reuse the existing guestInfo pattern
- Do NOT modify handleAddComment -- it already reads from guestInfo.name and guestInfo.email
