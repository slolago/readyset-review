The FolderCard component is a local function inside FolderBrowser.tsx. The FolderBrowser parent already manages `showReviewModal` state but it only holds a single folderId target. We need per-folder modal invocation.

**Approach:** Add an `onCreateReviewLink` optional prop to `FolderCard` that the parent wires up. The parent will track a `reviewModalFolderId` state separate from the existing `showReviewModal` / `folderId` state. When a folder's "Create review link" is clicked, it sets `reviewModalFolderId` to that folder's id and opens a second modal instance (or reuses the existing one by updating its target).

**Step-by-step changes to FolderBrowser.tsx:**

1. Add new state in `FolderBrowser` near the existing `showReviewModal` state:
   ```typescript
   const [folderReviewTarget, setFolderReviewTarget] = useState<string | null>(null);
   ```

2. In the JSX where `CreateReviewLinkModal` is already rendered (find `showReviewModal &&` usage), add a second conditional modal for folder-targeted creation:
   ```tsx
   {folderReviewTarget !== null && (
     <CreateReviewLinkModal
       projectId={projectId}
       folderId={folderReviewTarget}
       onClose={() => setFolderReviewTarget(null)}
     />
   )}
   ```

3. Add `onCreateReviewLink` to the `FolderCard` prop types:
   ```typescript
   onCreateReviewLink?: () => void;
   ```

4. In the `FolderCard` component body, add the `Link` icon import to the destructure at the top of FolderBrowser — `Link as LinkIcon` is already imported at line 23 in the file header imports. Use it in the dropdown item.
