# Plan 42-01: Compare view audio switching + comment panel

## Goal
Compare view lets users choose which side's audio they hear by clicking the panel header; shows that version's comments in a panel below.

## Context
- **File**: `src/components/files/AssetCompareModal.tsx`
- **Audio switching**: already exists via toggle button; now also wired to panel header clicks
- **Comment panel**: new — fetches comments for both assets on mount, shows active side's comments

## Implementation

### AssetCompareModal.tsx

**Props added:**
- `projectId?: string`
- `getIdToken?: () => Promise<string | null>`

**State added:**
- `commentsA: Comment[]`
- `commentsB: Comment[]`

**Effect added:**
- Fetches comments for both assets when `getIdToken` is provided

**Panel headers (side-by-side mode):**
- Made clickable — `onClick={() => setAudioSide('A'/'B')}`
- Visual indicator: Volume2 icon (active) / VolumeX icon (inactive)
- Active side gets `bg-frame-accent/10` tint

**Comments panel:**
- Rendered after controls bar when either asset has comments
- Shows `audioSide === 'A' ? commentsA : commentsB`
- Stable ref — switching sides just renders different array, no flicker
- Displays author initial avatar, name, timestamp, body

### FolderBrowser.tsx
- Pass `projectId` and `getIdToken` to `<AssetCompareModal>`

## Verification
1. `npm run build` — zero errors
2. Open compare modal for 2 assets with comments → comment panel appears
3. Click the other panel header → audio switches + comment panel updates
4. Comments load without flicker on side switch
