# Plan 27-01: Asset Comparison Modal

## Objective

Add a side-by-side asset comparison feature. When exactly 2 assets are selected in the grid, a "Compare" button appears in the multi-select action toolbar. Clicking it opens a full-screen modal with two synchronized video/image players. Play/pause and seek are shared; audio plays from one side at a time with a toggle. No additional API calls â€” the modal reuses signedUrl from the already-fetched asset objects.

## Context

- `src/components/files/FolderBrowser.tsx` â€” Hosts multi-select state (`selectedIds`) and the action bar at the bottom. This is where the Compare button goes.
- `src/components/files/AssetGrid.tsx` â€” Renders asset cards; passes `selectedIds` down.
- `src/types/index.ts` â€” `Asset` interface has `signedUrl` (via API response cast), `name`, `type`, `thumbnailUrl`.
- `src/components/files/AssetCompareModal.tsx` â€” New file to create.

## Tasks

### Task 1: Create AssetCompareModal component

**Type:** auto

**Done when:** `src/components/files/AssetCompareModal.tsx` exists, accepts two `Asset` objects, renders a full-screen modal with two side-by-side panels. Each panel shows asset name + a video or image element. A shared Play/Pause button controls both videos. A shared scrubber seeks both. An audio toggle switches which side has audio. Space bar toggles play/pause. X button closes modal.

**Implementation:**

1. Create `src/components/files/AssetCompareModal.tsx`.
2. Props: `assetA: Asset`, `assetB: Asset`, `onClose: () => void`.
3. Use `useRef` for two `<video>` elements: `videoARef`, `videoBRef`.
4. State: `isPlaying: boolean`, `currentTime: number`, `duration: number`, `audioSide: 'A' | 'B'`.
5. On mount, set initial muted state: videoA muted = (audioSide !== 'A'), videoB muted = (audioSide !== 'B').
6. Play/Pause handler: toggles `isPlaying`; calls `play()`/`pause()` on both refs.
7. Seek handler (scrubber `onChange`): sets `currentTime` on both video elements directly.
8. Audio toggle button: flips `audioSide`, updates `muted` on both refs.
9. `timeupdate` event listener on videoA â†’ updates `currentTime` state (for scrubber position).
10. `loadedmetadata` on videoA â†’ sets `duration` state.
11. Keyboard handler: `useEffect` attaches `keydown` listener; Space â†’ togglePlayPause; Escape â†’ onClose.
12. For image assets, render `<img>` instead of `<video>`; hide audio toggle when no video.
13. Overlay: fixed full-screen, z-50, dark bg `bg-black/95`.
14. Layout: two panels flex side-by-side, each `flex-1`, showing name header + player.
15. Bottom controls bar: scrubber (range input, full width), then play/pause + audio toggle buttons.
16. Use `GitCompare` or `Columns2` icon from lucide-react for the modal header (or just text "Compare").
17. Import `Asset` from `@/types`.

**Files:** `src/components/files/AssetCompareModal.tsx`

### Task 2: Add Compare button to FolderBrowser multi-select toolbar

**Type:** auto

**Done when:** FolderBrowser imports `AssetCompareModal`; when exactly 2 assets are selected the "Compare" button is enabled; when more than 2 or non-assets are selected it is disabled with tooltip text; clicking opens the modal with the two selected assets.

**Implementation:**

1. Import `AssetCompareModal` from `./AssetCompareModal`.
2. Import `Columns2` (or `GitCompare`) from `lucide-react`.
3. Add state: `const [showCompareModal, setShowCompareModal] = useState(false)`.
4. Derive selected assets: `const selectedAssets = assets.filter(a => selectedIds.has(a.id))`.
5. Derive: `const canCompare = selectedAssets.length === 2`.
6. In the multi-select action bar (the `fixed bottom-6` bar), add Compare button before Move:
   ```
   <button
     onClick={() => canCompare && setShowCompareModal(true)}
     disabled={!canCompare}
     title={canCompare ? 'Compare assets' : 'Select exactly 2 assets to compare'}
     className={...}
   >
     <Columns2 className="w-3.5 h-3.5" />
     Compare
   </button>
   ```
7. Render `AssetCompareModal` when `showCompareModal && canCompare`:
   ```
   {showCompareModal && canCompare && (
     <AssetCompareModal
       assetA={selectedAssets[0]}
       assetB={selectedAssets[1]}
       onClose={() => setShowCompareModal(false)}
     />
   )}
   ```
8. When modal closes, do NOT clear selectedIds (user may want to re-open).
