# Phase 62 Plan 01: Signed URL Caching

## Objective

Stop regenerating GCS signed URLs on every list request. Cache them on the
asset doc with expiry metadata; reuse while remaining TTL > 30 min; regenerate
and write back otherwise.

## Context

@.planning/phases/62-signed-url-caching/62-CONTEXT.md
@.planning/REQUIREMENTS.md (CACHE-01..03)

Hot paths that currently regenerate URLs on every request:
- `GET /api/assets` — 3 sign calls × N assets (main + thumbnail + sprite)
- `GET /api/review-links/[token]` — 2+ sign calls × N assets per guest load

A 200-asset review link was firing ~600 GCS signing calls per guest page load.

## Tasks

### Task 1 (auto) — CACHE-01 types + helper

Add to `Asset` interface: `signedUrl?`, `signedUrlExpiresAt?: Timestamp`,
`thumbnailSignedUrl?`, `thumbnailSignedUrlExpiresAt?: Timestamp`,
`spriteSignedUrl?`, `spriteSignedUrlExpiresAt?: Timestamp`.

Create `src/lib/signed-url-cache.ts` exporting `getOrCreateSignedUrl({
gcsPath, cached, cachedExpiresAt, ttlMinutes })` returning `{ url, fresh,
expiresAt }`. Returns cached when remaining TTL > 30 min; else regenerates.
Does NOT write back — caller owns batch-update timing.

**Verify:** `npx tsc --noEmit` passes.

### Task 2 (auto) — CACHE-01 wire into /api/assets

Replace the 3 parallel `generateReadSignedUrl` calls in the `Promise.all` at
route.ts:87-107 with `getOrCreateSignedUrl`. Collect assets that regenerated,
batch-commit `{ signedUrl, signedUrlExpiresAt, ... }` to Firestore before
returning. TTLs: main=120, thumbnail=720, sprite=720.

Download URL stays per-request (filename-dependent, not hot-path).

**Verify:** `npx tsc --noEmit` passes; client-visible fields (`signedUrl`,
`thumbnailSignedUrl`, `spriteSignedUrl`) unchanged in shape.

### Task 3 (auto) — CACHE-02 wire into review-link GET

Same treatment in `decorate` inside `/api/review-links/[token]/route.ts`.
Buffer pending writes at the outer scope of `GET`, flush before each terminal
`NextResponse.json()` (3 sites: folder view, array-based, legacy).

**Verify:** `npx tsc --noEmit` passes.

### Task 4 (auto) — CACHE-03 verify thumbnail/sprite TTL

Confirm Task 2/3 use 720-min TTL for thumbnail and sprite (less-sensitive
assets, longer TTL = fewer regenerations). Inspect only; no code change if
already correct.

## Success Criteria

- [x] Asset type gains 6 new cache fields
- [x] `getOrCreateSignedUrl` helper exists and typechecks
- [x] `/api/assets` reuses cached URLs when >30 min remaining
- [x] `/api/review-links/[token]` reuses cached URLs when >30 min remaining
- [x] Fresh URLs are written back to Firestore
- [x] Client-visible field names unchanged (`signedUrl`, `thumbnailSignedUrl`,
      `spriteSignedUrl`)
- [x] No admin UI for cache invalidation (out of scope)
- [x] `npx tsc --noEmit` clean after each task

## Anti-scope

- Don't change the shape the client sees (AssetCard reads `signedUrl`,
  `thumbnailSignedUrl`, `spriteSignedUrl` — preserve).
- No admin UI for cache invalidation.
- Don't touch the owner-only `/api/review-links/[token]/contents` route.
- Don't cache the download URL (filename disposition varies with rename).
