# Phase 61 Plan 01: Transactional Mutations

## Objective

Wrap stack-mutation endpoints (merge, unstack) and the upload auto-versioning scan in Firestore `runTransaction` so concurrent writes cannot corrupt version numbering. Also validate folder liveness during upload signed-url issuance.

## Context

- `@src/app/api/assets/reorder-versions/route.ts` (reference — already uses `db.runTransaction`)
- `@src/lib/version-groups.ts` (shared helper — needs a tx-aware variant)

## Tasks

### Task 1 — TXN-01: merge-version → runTransaction (type=auto)
Refactor `src/app/api/assets/merge-version/route.ts` from `db.batch()` to `db.runTransaction()`. All reads (both group members) happen first via `fetchGroupMembersTx` before any `tx.update()`. Add `fetchGroupMembersTx` to `src/lib/version-groups.ts`.

### Task 2 — TXN-02: unstack-version → runTransaction (type=auto)
Same treatment for `src/app/api/assets/unstack-version/route.ts`. `fetchGroupMembersTx` inside the tx, then all `tx.update()` writes.

### Task 3+4 — TXN-03 + TXN-04: signed-url transactional auto-version + folder live-check (type=auto)
Combined in one transaction: read folder doc (reject 404 if missing / `deletedAt` set), scan project assets for name collision, compute next version, write new asset doc. Concurrent uploads of the same `(projectId, folderId, filename)` serialize to correct V1, V2.

## Verification

- `npx tsc --noEmit` clean
- `npx vitest run` all green
- Grep: no `db.batch()` remains in the three target routes
- Grep: all three routes contain `runTransaction`

## Success Criteria

- 4 TXN requirements (TXN-01..04) implemented
- No API shape changes, no new error formats beyond `404 Folder not found` for TXN-04
- No UI touches
