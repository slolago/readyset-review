# Plan 40-01: Debug and fix showAllVersions propagation

## Goal
The "Show all versions" toggle on review link creation actually shows all versions on the review page.

## Root Cause
`src/app/api/review-links/[token]/route.ts` lines 95-105: `showAllVersions` flag is stored on the Firestore doc but never read. The version-grouping logic always called `sorted[0]` (latest only), regardless of the flag.

## Fix
After grouping assets by `versionGroupId`, branch on `link.showAllVersions`:
- `true`: flatMap each group sorted newest-first — every version becomes its own card
- `false` (default): keep existing behavior — only latest version per group

## Verification
1. `npm run build` — zero errors
2. Create review link with "Show all versions" ON → opening that link shows all versions as individual cards (stack of 2 → 2 cards)
3. Create review link with "Show all versions" OFF → only latest version shown per stack

## Files Changed
- `src/app/api/review-links/[token]/route.ts` — branch on `link.showAllVersions` in grouping logic
