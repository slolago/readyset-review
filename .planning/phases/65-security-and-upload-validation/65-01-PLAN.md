# Phase 65 Plan 01: Review-link password hashing + upload validation

## Objective

Harden the review-link + upload pipeline: bcrypt-hash passwords with transparent legacy migration, move password submission off the query string, and add server-side MIME validation on upload completion.

## Tasks

### Task 1 — SEC-20: bcrypt hashing + legacy plaintext migration
- Install `bcryptjs` + `@types/bcryptjs` (pure JS, Vercel-safe).
- Add `hashPassword(plain)` and `verifyPassword(plain, stored)` helpers to `src/lib/review-links.ts`.
- `verifyPassword` returns `{ ok, needsUpgrade }`; `needsUpgrade=true` means stored value was plaintext and a re-hash should be persisted.
- Hash at write on `POST /api/review-links` and `PATCH /api/review-links/[token]`.
- `assertReviewLinkActive` uses verifyPassword and exposes `needsUpgrade` via a returned info object; callers fire-and-forget the upgrade.

### Task 2 — SEC-21: password via header, deprecate query param
- Accept `x-review-password` request header on `GET /api/review-links/[token]` and `GET /api/comments?reviewToken=...`.
- Still accept `?password=` for back-compat; log a deprecation warning when used.
- Update client `fetchReview` and comment fetches to send the header instead of the query string.

### Task 3 — SEC-22: verify already in place
- Confirm `upload/complete` already calls `verifyGcsObject` (OBS-04).

### Task 4 — SEC-23: MIME validation on upload/complete
- Extend `verifyGcsObject` (or add sibling) to return `contentType`.
- On `/api/upload/complete`, reject when reported Content-Type is not on `ACCEPTED_MIME` allow-list. Fall back to stored `asset.mimeType` when GCS reports absent/octet-stream.

## Success Criteria

- New passwords are stored as bcrypt hashes (start with `$2a$` / `$2b$`).
- Plaintext legacy passwords verify once, then get silently rewritten as hashes.
- Clients no longer put passwords in the URL; header path works; query path still works with a deprecation log.
- Upload complete rejects 400 when GCS content-type is not on the allow-list.
- `npm run typecheck` passes. `npm test` passes.

## Anti-scope

No rate limiting, no CAPTCHA, no password-complexity rules, no session management changes.
