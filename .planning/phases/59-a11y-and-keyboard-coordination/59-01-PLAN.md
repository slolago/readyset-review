  <name>Task 1 (A11Y-shared): Create useFocusTrap + useModalOwner hooks</name>
  <files>src/hooks/useFocusTrap.ts, src/hooks/useModalOwner.ts</files>
  <action>
Create two tiny hooks. No external deps beyond React.

**src/hooks/useFocusTrap.ts**
- Export `useFocusTrap(containerRef: React.RefObject<HTMLElement>, active: boolean)`.
- On mount (when `active` becomes true):
  1. Remember `document.activeElement` as `previouslyFocused`.
  2. Query focusable elements inside `containerRef.current`:
     selector = `'a[href], area[href], input:not([disabled]):not([type="hidden"]), select:not([disabled]), textarea:not([disabled]), button:not([disabled]), iframe, object, embed, [tabindex]:not([tabindex="-1"]), [contenteditable]'`.
     Filter to visible (offsetParent !== null).
  3. Focus the first focusable (or the container itself with tabIndex=-1 if none).
- Attach a `keydown` listener to `containerRef.current` (or document if container ref is null) that only handles `Tab`:
  - Recompute the focusable list on each Tab (DOM may have changed).
  - If focusables.length === 0, preventDefault and stop.
  - Shift+Tab on first → focus last, preventDefault.
  - Tab on last → focus first, preventDefault.
- On cleanup: restore focus to `previouslyFocused` if it still exists in DOM and is focusable.
- Do nothing when `active` is false.

**src/hooks/useModalOwner.ts**
- Module-scope `let count = 0;`
- Export `useModalOwner(active: boolean)`.
- Effect keyed on `active`:
  - When `active` becomes true: `count++`; if count === 1, set `document.body.dataset.modalOpen = 'true'`.
  - On cleanup (or `active` becoming false): `count--`; if count === 0, delete `document.body.dataset.modalOpen` (use `delete document.body.dataset.modalOpen`).
- Guard against count going negative (clamp to 0 defensively in cleanup).

No tests needed — these get exercised by the downstream integration tasks and the phase verification step.
  </action>
