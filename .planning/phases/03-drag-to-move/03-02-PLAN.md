# Plan 03-02: Folder Drop Targets (Drag-to-Move)

## Objective
Add drop-target behavior to FolderCard components so selected items can be dragged and dropped directly onto a folder to move them.

## Context
- FolderBrowser.tsx has handleMoveSelected() which calls the move API
- AssetGrid and FolderCard items are selectable (selectedIds Set)
- Phase 03-01 adds drag source (draggable attribute + dataTransfer payload)
- This plan adds the drop target side: folder cards highlight on hover and accept drops

## Tasks

### Task 1: Add dragOverFolderId state to FolderBrowser
type: auto

Add `dragOverFolderId` state (string | null) to FolderBrowser.
Wire onFolderDragOver, onFolderDragLeave, onFolderDrop handlers.
Pass them down to each FolderCard via props.

Done criteria: State declared, handlers defined, FolderCard receives drag props.

### Task 2: Add drag drop props to FolderCard
type: auto

FolderCard accepts isDropTarget, onDragOver, onDragLeave, onDrop props.
Apply ring highlight when isDropTarget is true.
onDragOver calls e.preventDefault() and sets parent state.
onDragLeave clears parent state.
onDrop decodes application/x-frame-move payload and calls move handler.
Self-drop prevention: if payload contains the folder's own ID, do nothing.

Done criteria: FolderCard visually highlights on drag-over, drops trigger move API call.

## Verification
- TypeScript compiles clean (npx tsc --noEmit)
- Dragging selected items onto a folder triggers handleMoveSelected with that folder's ID
- Dragging a folder onto itself does nothing
- Drop highlight ring appears on hover and disappears on leave/drop
