**FolderBrowser component — drag start handler:**

3. After the `toggleSelect` callback (around line 224), add a new handler:
   ```typescript
   const handleItemDragStart = useCallback((itemId: string, e: React.DragEvent) => {
     // If the dragged item is already selected, carry all selected items.
     // Otherwise carry only this one item.
     const ids = selectedIds.has(itemId) ? Array.from(selectedIds) : [itemId];
     e.dataTransfer.setData('application/x-frame-move', JSON.stringify({ ids }));
     e.dataTransfer.effectAllowed = 'move';
   }, [selectedIds]);
   ```

   Use MIME type `application/x-frame-move` (not `text/plain`) so that the container's OS-file-drop handler can distinguish internal item drags from OS file drops.

4. In the `handleContentMouseDown` handler (line 208), the existing early return `if ((e.target as HTMLElement).closest('[data-selectable]')) return;` already prevents rubber-band from starting on cards. This is sufficient — no changes needed.

**Wire handlers to cards:**

5. In the folders grid (line 560), pass `onDragStart` to each `<FolderCard>`:
   ```typescript
   onDragStart={(e) => handleItemDragStart(folder.id, e)}
   ```

6. In the `<AssetGrid>` usage (line 581), pass the new prop:

   ```typescript
   onAssetDragStart={handleItemDragStart}
   ```

**Guard the container drop handler:**

7. In `handleDrop` (line 361), add an early return at the top to ignore internal drags (they will be handled by folder card drop zones in Plan 03-02):

   ```typescript
   // Ignore internal item-move drags — handled by folder drop zones
   ```
