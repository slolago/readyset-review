4. Add a `commitRename` async function:
   ```typescript
   const commitRename = async () => {
     const trimmed = renameValue.trim();
     if (!trimmed || trimmed === asset.name) {
       setIsRenaming(false);
       return;
     }
     try {
       const token = await getIdToken();
       const res = await fetch(`/api/assets/${asset.id}`, {
         method: 'PUT',
         headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${token}` },
         body: JSON.stringify({ name: trimmed }),
       });
       if (res.ok) {
         toast.success('Renamed');
         onDeleted?.(); // reuse the refresh callback — it causes the parent to refetch
       } else {
         toast.error('Rename failed');
       }
     } catch {
       toast.error('Rename failed');
     } finally {
       setIsRenaming(false);
     }
   };
   ```
   NOTE: `onDeleted` is the existing callback the parent uses to trigger refetch after mutation. Reusing it is correct here — it causes AssetGrid/FolderBrowser to call refetchAssets. If a dedicated `onRenamed` prop is preferred, add it alongside `onDeleted` in the interface and call it instead.

5. In the `<div className="p-3">` info section, replace the static `<p>` for asset.name with a conditional:
   ```tsx
   {isRenaming ? (
     <input
       ref={renameInputRef}
       className="w-full bg-frame-bg border border-frame-accent rounded px-1.5 py-0.5 text-sm font-medium text-white outline-none focus:ring-1 focus:ring-frame-accent"
       value={renameValue}
       onChange={(e) => setRenameValue(e.target.value)}
       onKeyDown={(e) => {
         if (e.key === 'Enter') { e.preventDefault(); commitRename(); }
         if (e.key === 'Escape') { setIsRenaming(false); }
       }}
       onBlur={commitRename}
       onClick={(e) => e.stopPropagation()}
     />
   ) : (

       {asset.name}

   )}
   ```
