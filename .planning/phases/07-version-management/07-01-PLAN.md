Add the `VersionStackModal` function component at the bottom of AssetCard.tsx, after `AssetFolderPickerModal`. This component is intentionally co-located (same pattern as AssetFolderPickerModal).

Props interface:
```typescript
interface VersionStackModalProps {
  asset: Asset;
  onClose: () => void;
  onDeleted?: () => void;
  getIdToken: () => Promise<string | null>;
}
```

Behavior:
- On mount, fetch GET /api/assets/{asset.id} with Bearer token. The response shape is `{ asset, versions: AssetWithUrl[] }`. Store versions in local state. Show a loading spinner while fetching.
- Render a modal with the same backdrop + card pattern as AssetFolderPickerModal (`fixed inset-0 z-50 flex items-center justify-center bg-black/60 backdrop-blur-sm`). Max width: `max-w-md`. Click outside closes.
- Header: "Version stack" title (left) + X button (right) — same as AssetFolderPickerModal's header.
- Body: scrollable list (`max-h-80 overflow-y-auto`). For each version in the fetched list, render a row:
  - Left: version label "V{version.version}" in a small rounded badge using `bg-frame-accent/20 text-frame-accent text-xs px-2 py-0.5 rounded font-mono`
  - Middle: two lines — `version.name` truncated (text-sm text-white), and below it the upload date formatted as `new Date(version.createdAt.toDate()).toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })` plus `· ${version.uploadedBy}` (text-xs text-frame-textMuted)
  - Right: a delete button — `<Trash2 className="w-4 h-4" />` in red, only shown if versions.length > 1 OR if this is not the only version left. Specifically: disable/hide the delete button when versions.length === 1 (cannot delete the last version).
- Delete flow for a version row:
  1. Call `confirm('Delete version V{N}?')`. If cancelled, stop.
  2. Fetch DELETE /api/assets/{version.id} with Bearer token.
  3. On success: remove the version from local state. If the deleted version was the currently-viewed asset (version.id === asset.id) and other versions remain, call `onDeleted?.()` to refresh the parent grid and close the modal. If local state now has 0 versions left, call `onDeleted?.()` and close the modal.
  4. Show `toast.success('Version deleted')` on success, `toast.error('Delete failed')` on failure.
- Footer: a single "Close" button (`w-full mt-2 py-2 text-sm text-frame-textMuted hover:text-white transition-colors`).

Add these imports to the top of the file if not already present: `Trash2` (already imported), `Layers` (already imported), `X` (already imported). No new imports needed.

The `toDate()` call on `version.createdAt` handles Firebase Timestamps. If `version.createdAt` is already a plain object from the API (serialized), guard with: `typeof version.createdAt?.toDate === 'function' ? version.createdAt.toDate() : new Date(version.createdAt?._seconds * 1000 || Date.now())`.
