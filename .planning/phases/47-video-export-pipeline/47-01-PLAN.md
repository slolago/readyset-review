Purpose: Requirements EXPORT-01, EXPORT-02, EXPORT-03 — a trusted trim/convert pipeline so reviewers can share snippets without leaving the app.

Output: ExportJob type, exports Firestore collection, POST/GET /api/exports endpoints, ExportModal UI wired into VideoPlayer, outputFileTracingIncludes updated for Vercel bundling.

  <name>Task 2: POST /api/exports — MP4 path (copy + H.264/AAC fallback)</name>
  <files>src/app/api/exports/route.ts</files>
  <action>
Create POST handler. Structure:

1. Authenticate via `getAuthenticatedUser`. Parse body `{ assetId, format, inPoint, outPoint, filename }`. Validate:
   - format must be 'mp4' or 'gif'
   - inPoint >= 0, outPoint > inPoint, (outPoint - inPoint) <= 120 (cap at 2 min to fit maxDuration=60 with headroom)
   - filename non-empty, strip path chars (`/[^a-zA-Z0-9._ -]/g` → '_')

2. Load asset via `getAdminDb().collection('assets').doc(assetId).get()`. Check project membership via `canProbeAsset(user, project)` (same perm gate as probe; it represents "can read this asset's media").

3. Call `createExportJob(...)` — get jobId. Respond 202 with `{ jobId }` immediately? NO — per design we run ffmpeg inline and only return when done. Set `maxDuration = 60`. Update job status to 'encoding' before spawn.

4. Generate a fresh signed read URL for `asset.gcsPath` (60 min). Resolve ffmpeg via `resolveFfmpeg()`. If null, mark job failed and return 500.

5. MP4 branch (this task):
   - Determine if stream-copy is safe. Safe if: `asset.videoCodec === 'h264'` AND (`asset.audioCodec === 'aac' || !asset.audioCodec`) AND container covers mp4 (`asset.containerFormat?.includes('mp4')` OR format unknown).
   - Output path: `path.join(os.tmpdir(), \`export-${jobId}.mp4\`)`.
   - Copy args:
     `['-y', '-ss', String(inPoint), '-i', signedUrl, '-t', String(outPoint - inPoint), '-c', 'copy', '-movflags', '+faststart', outPath]`
   - Re-encode args:
     `['-y', '-ss', String(inPoint), '-i', signedUrl, '-t', String(outPoint - inPoint), '-c:v', 'libx264', '-preset', 'fast', '-crf', '23', '-c:a', 'aac', '-movflags', '+faststart', outPath]`
   - Run via `spawn(ffmpegPath, args)` pattern from probe/route.ts. On non-zero exit → if copy was attempted, retry once with re-encode args (common: timestamps don't align on keyframes). On second failure, mark job failed with stderr snippet and return 500.

6. Read the output file to a Buffer via `await fs.promises.readFile(outPath)`. GCS path: `exports/${user.id}/${jobId}.mp4`. Upload via `uploadBuffer(gcsPath, buf, 'video/mp4')`. Delete local temp file.

7. Update job: `status: 'ready', gcsPath, completedAt: FieldValue.serverTimestamp()`. Return `{ jobId, status: 'ready' }`.

8. Defer GIF branch to Task 3 — for now, if `format === 'gif'` return 501 `{ error: 'GIF not yet implemented' }` (Task 3 will replace this).

9. Also add a `GET` export in this same file: lists current user's recent jobs via `listUserExports(user.id, 20)`. Each returned job gets a fresh signed download URL when status === 'ready' (via `generateDownloadSignedUrl(job.gcsPath, filename + '.' + format, 60)`) attached as `.signedUrl` (transient, not stored).

Set at top of file:
```typescript
export const runtime = 'nodejs';
export const maxDuration = 60;
```

Anti-scope: no job queue, no BullMQ, no Vercel background functions. Inline spawn, synchronous-from-client's-perspective. Polling is via GET /api/exports/[jobId] (Task 4).

Addresses EXPORT-01 (submit), EXPORT-02 (MP4 codec handling). GIF (EXPORT-02) handled in Task 3.

<task type="auto" tdd="false">
  <name>Task 3: POST /api/exports — GIF path (two-pass palette)</name>
  <files>src/app/api/exports/route.ts</files>
  <action>
Extend the POST handler from Task 2 with the GIF branch, replacing the 501 stub.

For `format === 'gif'`:

1. Temp paths:
   - `palettePath = path.join(os.tmpdir(), \`palette-${jobId}.png\`)`
   - `outPath = path.join(os.tmpdir(), \`export-${jobId}.gif\`)`

2. Pass 1 (palettegen):
   ```
   ['-y', '-ss', String(inPoint), '-i', signedUrl, '-t', String(outPoint - inPoint),
    '-vf', 'fps=15,scale=720:-1:flags=lanczos,palettegen=stats_mode=diff',
    palettePath]
   ```

3. Pass 2 (paletteuse):
   ```
   ['-y', '-ss', String(inPoint), '-i', signedUrl, '-t', String(outPoint - inPoint),
    '-i', palettePath,
    '-filter_complex', 'fps=15,scale=720:-1:flags=lanczos[x];[x][1:v]paletteuse=dither=bayer:bayer_scale=5',
    '-loop', '0',
    outPath]
   ```

4. Run each pass via the same `spawn` helper. If either pass exits non-zero, mark job failed with stderr tail and 500. Always clean up palettePath in finally.

5. Upload via `uploadBuffer(\`exports/${user.id}/${jobId}.gif\`, buf, 'image/gif')`. Delete both temp files.

6. Same ready-state update as MP4 branch. Final filename on download: `${sanitizedFilename}.gif`.

Notes:
- `-loop 0` makes the GIF loop forever (EXPORT-02 "looping")
- fps=15 and scale=720:-1 keep size reasonable for typical review clips
- `stats_mode=diff` optimizes palette for motion; `dither=bayer:bayer_scale=5` controls banding without exploding file size
- Source aspect ratio is preserved (-1 height)

No public API shape change — same request body, same response.
