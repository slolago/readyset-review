# Plan 39-01: Remove "copy of" prefix from copy API

## Goal
Copying an asset or folder preserves the original name — no "copy of" prefix added.

## Context
- **Files**: `src/app/api/assets/copy/route.ts`, `src/app/api/folders/copy/route.ts`
- `assets/copy`: line 52 used `` `Copy of ${ver.name}` `` as default name
- `folders/copy`: line 37 used `` `Copy of ${source.name}` `` as default name

## Implementation

### assets/copy/route.ts
Change:
```ts
name: ver.id === assetId ? (name ?? `Copy of ${ver.name}`) : ver.name,
```
To:
```ts
name: ver.id === assetId ? (name ?? ver.name) : ver.name,
```

### folders/copy/route.ts
Change:
```ts
name: name ?? `Copy of ${source.name}`,
```
To:
```ts
name: name ?? source.name,
```

## Verification
1. `npm run build` — zero errors
2. Copy an asset to another folder → copy name matches original, no prefix

- `src/app/api/assets/copy/route.ts`
- `src/app/api/folders/copy/route.ts`
